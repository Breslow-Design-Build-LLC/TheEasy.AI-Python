- warnings (should confirm with customer)

## Audit Rules (apply in this order)

### A) Hard Blockers
Read from `audit_inputs` sub-objects:

1) If any gate output contains `missing_skus` with items:
   - add a blocker: "Missing SKU(s): <list>"
2) If any gate output contains `missing_pricing` with items:
   - add a blocker: "Missing price(s): <list>"
3) Read `flags.install_scope` and `flags.rep_override_required` from the context.
   If install_scope="supply_only" AND rep_override_required is not true:
   - add blocker: "Supply-only selected but rep override flag is missing."

### B) Dependency Checks (Warnings)
Read from the relevant gate outputs inside `audit_inputs`:

4) If s06_lighting_fans shows LED strip selected AND control box/driver not included:
   - warning: "LED strip selected — Control Box/Driver required."
5) If s07_heaters shows heaters selected with heater_qty >= 1:
   - If heater beams not included → warning: "Heaters selected — heater beams may be required (check heater rules)."
   - If heater controls not included → warning: "Heaters selected — controls may be required (check heater rules)."
6) If s08_shades_privacy shows shades selected with shade_qty >= 1 AND s10_electrical shows "E" in selected_letters:
   - If shade_electrical_up_to_2_qty is null → warning: "Shade electrical selected — missing computed electrical quantities."

### C) Confirmation Prompts (Rep-facing)
Read from `flags` in the context:

7) If `flags.orientation_review_required` is true:
   - confirmation: "Orientation impacts bay count. Confirm louver span direction and whether to swap or compare."
8) If `flags.comparison_mode` is true:
   - confirmation: "Comparison mode enabled. Ensure both options are priced and presented clearly."

### D) Override Notes
9) If `flags.rep_override_required` is true:
   - override_notes MUST include exactly: "Rep override required — document approval before sending quote."
