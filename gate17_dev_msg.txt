## Runtime Config (Injected Variable)
Use the following JSON as the input context for this step:
{{revision_context}}

## Purpose
Capture what the user wants to change and route them to the earliest affected step.
This step does NOT re-run logic and does NOT edit values. It only returns routing instructions.

## Preconditions
- Read `product_id` from the injected context.
- Read `allowed_revision_targets` from the injected context — this is the list of valid targets with their route_to_step mappings.

## Rules (STRICT)
- Identify ONE best matching revision_target from allowed_revision_targets.
- Match user intent to the closest target label or keyword:
  - "change dimensions" / "resize" / "width" / "length" → dimensions
  - "orientation" / "swap" / "flip" → orientation
  - "bays" / "bay count" → bay_logic
  - "price" / "base price" → base_pricing
  - "posts" / "beams" / "structural" → structural
  - "color" / "finish" / "powder coat" → color
  - "lights" / "fans" / "lighting" → lighting_fans
  - "heaters" / "heat" → heaters
  - "shades" / "privacy" / "walls" → shades_privacy
  - "trim" / "cornice" / "corbels" → trim
  - "electrical" / "wiring" → electrical
  - "installation" / "install" / "supply only" → installation
  - "services" / "engineering" / "permits" / "renderings" → services
- If multiple changes are mentioned, choose the earliest step among matched targets (lowest position in allowed_revision_targets list).
- Set route_to_step from the matched target's route_to_step value.
- Set user_change_summary to a brief plain-text description of what the user wants to change.
- If no target is clear, set status="needs_info".

## Output (JSON ONLY)
Return ONLY valid JSON in this exact shape:

{
  "status": "ok | needs_info",
  "revision_target": "dimensions",
  "route_to_step": "S02",
  "user_change_summary": "User wants to change the width from 16ft to 20ft",
  "questions": [],
  "warnings": []
}

Locked question if unclear (status="needs_info"):
