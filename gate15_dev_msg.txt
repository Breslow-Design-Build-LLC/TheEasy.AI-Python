## Output Rules (STRICT)

1) Determine FINAL_DIMENSIONS source:
   Read `comparison_mode` and `orientation_choice` from the injected context.
   Read `step_outputs.s02_dimensions` for dimension data.

   A) If comparison_mode is true:

      - include both options (keep + swap) from s02_dimensions
      - confirmed_width_ft and confirmed_length_ft MUST be null

   B) Else if orientation_choice is "keep_as_entered" or "swap":

      - confirmed_width_ft = s02_dimensions.width_ft_confirmed
      - confirmed_length_ft = s02_dimensions.length_ft_confirmed

   C) Else:
      - final_dimensions.mode="assumed"
      - confirmed_width_ft = s02_dimensions.dim_a_ft_rounded
      - confirmed_length_ft = s02_dimensions.dim_b_ft_rounded

2) Determine FINAL_BAY_LOGIC:
   Read from `step_outputs.s03_bay_logic`.

   A) If comparison_mode is true → use result_keep and result_swap
   B) Else → use result_single

3) AUDIT must be copied from `step_outputs.s14_audit` exactly.

4) STATUS:
   - If s14_audit.blockers length > 0 → final_status="blocked"
   - Else if s14_audit.warnings length > 0 OR s14_audit.confirmations length > 0 → final_status="needs_review"
   - Else → final_status="ready"

5) PRICING_OUTPUTS:
   - Collect all pricing-related data from step_outputs (s04 through s12) as-is.
   - Do NOT recalculate or modify any values.
